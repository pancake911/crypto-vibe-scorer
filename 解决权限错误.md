# 🔧 解决 EPERM 权限错误

## 问题原因

这个错误通常是因为：
1. 终端无法访问当前目录
2. 文件夹权限问题
3. 终端会话问题

## ✅ 解决方法（按顺序尝试）

### 方法一：重新打开终端并进入文件夹（最简单）

1. **完全关闭终端**
   - 关闭所有终端窗口

2. **重新打开终端**
   - Mac：按 `Command + 空格键`，输入"终端"，回车

3. **重新进入文件夹**
   ```bash
   cd ~/Desktop/cursor/crypto-vibe-scorer
   ```

4. **检查是否成功**
   ```bash
   pwd
   ```
   - 应该显示：`/Users/mima0000/Desktop/cursor/crypto-vibe-scorer`

5. **再次尝试安装**
   ```bash
   npm install
   ```

---

### 方法二：使用完整路径

如果方法一不行，尝试：

1. **在终端输入：**
   ```bash
   cd /Users/mima0000/Desktop/cursor/crypto-vibe-scorer
   ```

2. **然后：**
   ```bash
   npm install
   ```

---

### 方法三：检查并修复权限

1. **检查文件夹权限**
   ```bash
   ls -la ~/Desktop/cursor/crypto-vibe-scorer
   ```

2. **如果权限有问题，修复权限**
   ```bash
   chmod 755 ~/Desktop/cursor/crypto-vibe-scorer
   ```

3. **重新进入文件夹**
   ```bash
   cd ~/Desktop/cursor/crypto-vibe-scorer
   ```

4. **再次尝试**
   ```bash
   npm install
   ```

---

### 方法四：在Finder中打开终端

1. **打开Finder**
2. **找到文件夹**：`Desktop → cursor → crypto-vibe-scorer`
3. **右键点击文件夹**
4. **选择"服务" → "新建位于文件夹位置的终端窗口"**
   - 如果没有这个选项，选择"打开终端"
5. **终端会自动进入这个文件夹**
6. **直接输入：**
   ```bash
   npm install
   ```
